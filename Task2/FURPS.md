| F   | **Функциональные (Functionality)** |              |
| F1  | Сайт: клиент видит список доступных депозитов с актуальными ставками [ASR] | Источник ставок должен быть централизованным, не XLS. |
| F2  | Сайт: клиент может подать заявку на депозит, указав номер телефона и Ф.И.О. [ASR] | Чувствительная информация передаётся с сайта, требуется шифрование трафика. Заявка передаётся в систему кол-центра. |
| F3  | Кол-центр: менеджер видит заявку с сайта и может предложить клиенту особые условия | Менеджер изучает заявку в системе кол-центра и звонит клиенту. |
| F4  | Новый клиент после подачи заявки с сайта должен прийти в отделение для идентификации | Банки не могут принимать новых клиентов без личной проверки документов (KYC). |
| F5  | ИБ: клиент видит список доступных депозитов с актуальными ставками и персонализированные ставки лично для него [ASR] | Персонализация по данным клиента из АБС или процесса согласования. |
| F6  | ИБ: клиент может подать заявку на открытие депозита, указав счёт и сумму депозита [ASR] | Заявка создаётся без прямого вызова API АБС для снижения нагрузки на систему. |
| F7  | ИБ: операция открытия депозита подтверждается СМС-кодом | Функционал работы с СМС реализован в ядре системы, требует доработки подрядчика, но этого лучше избежать. Реализовать можно силами команды разработки банка. |
| F8  | Бэк-офис депозитов: менеджер обрабатывает заявку на открытие депозита в АБС (MVP) | Менеджер подтверждает условия депозита в АБС банка. |
| F9  | Клиенту приходит СМС-уведомление после подтверждения размера ставки и открытия депозита | Уведомления через СМС-шлюз на основе статусов заявки. |
| F10 | Централизованное управление ставками по депозитам: ввод и изменение ставок сотрудниками бэк-офиса депозитов и бэк-офиса кредитов [ASR] | Ставки сейчас ведутся только в XLS-файлах. Необходимо хранилище ставок в банковской системе, не XLS. Возможно реализовать в АБС. Требуется аудит изменений. |
| F11 | Сайт: клиент видит предложение открыть кредит по некоторой ставке, полная информация об условиях кредита хранится в Кредитном конвейере [ASR] | Интеграция с Кредитным конвейером для получения актуальных условий. |
| F12 | Сайт: клиент может подать заявку на кредит, указав номер телефона и Ф.И.О. Опционально может указать паспортные данные [ASR] | При указании паспортных данных предполагается сразу выдать решение об одобрении или неодобрении через систему скоринга. |
| F13 | Сайт: при указании паспортных данных клиенту сразу выдаётся решение об одобрении или неодобрении кредита | Для нового клиента используются только данные бюро кредитных историй (БКИ) для скоринга, так как внутренних данных банка ещё нет. |
| F14 | Клиенту приходит СМС после подачи заявки на кредит с сайта о том, что нужно прийти в отделение для получения кредита и идентификации | KYC офлайн для новых клиентов. |
| F15 | Отделение: менеджер может подать заявку на кредит через АБС. Если клиент уже получил предварительное одобрение на сайте, сотрудник должен продолжить работать с той же заявкой [ASR] | Требуется консистентность заявок между сайтом, отделением и интернет-банком. Для полностью нового клиента создаётся новая заявка. |
| F16 | ИБ: клиент видит список доступных кредитов с актуальными условиями, аналогично сайту. Также может выводиться предодобренное предложение по кредиту [ASR] | Предодобренное предложение означает, что по клиенту уже произошёл скоринг и была рассчитана приемлемая сумма кредита. |
| F17 | ИБ: клиент может подать заявку на кредит, указав счёт зачисления средств и данные для подачи заявки. Операция подтверждается СМС-кодом | Для клиента банка; предодобренные предложения проходят упрощённый маршрут обработки. |
| F18 | При подаче заявки на кредит в ИБ происходит повторный скоринг по упрощённой процедуре, если ситуация изменилась | Предодобренные заявки обрабатываются сотрудниками быстрее. |
| F19 | Клиенту приходит СМС-уведомление об изменении статуса заявки на кредит | Информирование клиента через СМС-шлюз на основе статусов заявки. |
| U   | **Удобство использования (Usability)** |              |
| U1  | Интерфейс работы должен быть максимально удобным для клиента | Бизнес ожидает высокий уровень удобства использования. |
| U2  | Соблюдение системы дизайна для приложений, принятой в компании | Пользователи должны видеть привычные цвета и брендированные элементы. |
| R   | **Надёжность (Reliability)**         |              |
| R1  | Все сервисы должны работать 24/7 и быть доступны в 99,9% случаев [ASR] | Требование доступности для всех сервисов системы. |
| R2  | В случае сбоев в ЦОД интернет-банк должен быть доступен и выдерживать требуемую нагрузку, переключившись на резервный ЦОД [ASR] | Отказоустойчивость на уровне инфраструктуры. Команда интернет-банка утверждает, что требования пока невыполнимы из-за малого количества клиентов, но бизнес требует выполнения. |
| R3  | Избежать прямой работы интернет-банка с API АБС в новом процессе [ASR] | База данных АБС уже перегружена. Онлайн-подача заявок на большое количество продуктов может поставить под угрозу работоспособность банка. |
| R4  | Чувствительная информация, передаваемая с сайта, должна быть защищена механизмом шифрования трафика [ASR] | Требование безопасности для защиты данных клиентов. |
| R5  | Данные в интернет-банке при передаче должны быть защищены механизмом шифрования [ASR] | Требование безопасности для защиты данных клиентов. |
| P   | **Производительность (Performance)**   |              |
| P1  | Отклик по всем операциям должен быть максимально быстрым и занимать миллисекунды | Бизнес ожидает высокую скорость отклика системы. |
| P2  | Ускорить загрузку справочных данных при проведении платежей, чтобы операции занимали меньше времени [ASR] | Сейчас справочные данные загружаются больше секунды, что создаёт проблему для пользователей. |
| P3  | Предусмотреть равномерное горизонтальное масштабирование и распределение запросов между серверами, приложениями и ЦОД [ASR] | Требование масштабируемости системы. АБС может масштабироваться только вертикально из-за своей базы данных. |
| P4  | Система кредитного скоринга не должна нагружаться дополнительными процессами по предрасчётам скорингов | Система очень важна для работы банка и испытывает повышенную нагрузку в рабочие часы. |
| P5  | Кредитный конвейер и система кредитного скоринга должны поддерживать горизонтальное масштабирование и работу 24/7 | Системы разработаны с учётом возможного роста нагрузки. |
| S   | **Поддерживаемость (Supportability)**  |              |
| S1  | Разработка документации для дальнейшего расширения системы | Требование для обеспечения поддерживаемости и развития системы. |
| S2  | Использовать технологии, которые уже есть в банке, например, базы данных MS SQL и Oracle [ASR] | Снижение рисков внедрения и использование существующей экспертизы. |
| S3  | Новые технологии должны быть совместимы с существующими платформами разработки [ASR] | Обеспечение совместимости при внедрении новых решений. |
| S4  | Желательно, чтобы внутри банка уже была экспертиза по новым технологиям на уровне языков программирования | Снижение рисков внедрения за счёт использования знакомых технологий. |
| +R  | **+ Ограничения (Restrictions)**       |              |
| +R1 | Ставки по депозитам сейчас ведутся только в XLS-файлах [ASR] | Текущее состояние системы. Требуется централизация ставок в банковской системе. |
| +R2 | С функционалом работы с СМС в ядре системы лучше избежать доработок со стороны подрядчика | Функционал реализован в ядре системы и требует доработки подрядчика. Реализовать бизнес-функционал можно силами команды разработки банка. |
| +R3 | Текущая версия платформы интернет-банка несовместима с Kafka [ASR] | Ограничение платформы интернет-банка. |
| +R4 | Возможно стоит подумать о переводе интернет-банка на микросервисную архитектуру, но пока только в рамках задачи открытия депозитов [ASR] | Текущий интернет-банк — монолит. Рассматривается переход на микросервисную архитектуру в ограниченном объёме. |
| +R5 | Если нужны очереди сообщений, то лучше использовать Kafka на перспективу | Kafka предпочтительна для очередей сообщений, но есть ограничения по совместимости с текущей версией интернет-банка. |
| +R6 | АБС может масштабироваться только вертикально из-за своей базы данных [ASR] | Ограничение масштабируемости АБС. Нельзя нагружать онлайн-интеграциями. |
| +R7 | Обмен между АБС и Кредитным конвейером происходит раз в сутки, ускорить его чаще невозможно | Ограничение частоты обмена данными между системами. |
| +R8 | Ожидается, что заявки на кредит по предодобренным предложениям будут обрабатываться сотрудниками в течение рабочего дня | Бизнес-ожидание, несмотря на ограничение обмена АБС↔Кредитный конвейер раз в сутки. |
| +R9 | Для нового клиента при скоринге можно использовать только данные бюро кредитных историй (БКИ) | Ограничение по доступным данным для скоринга новых клиентов до прохождения KYC в отделении. |
